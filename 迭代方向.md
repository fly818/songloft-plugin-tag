**Songloft 插件迭代优化蓝图**：

---

### 🎯 一、 核心定位：纯粹的"元数据治愈中心"

*   **专注数据清洗**：假设音频文件已经通过其他途径进入 Songloft 媒体库，插件只负责"望闻问切"——找出错误标签、补全缺失的封面/歌词/ISRC，并提供优雅的 UI 让用户进行最终校对。
*   **非侵入式体验**：所有操作均在 Songloft 数据库和插件本地缓存中进行，给予对文件目录结构有强迫症的高级玩家绝对的掌控感。

---

### 🧠 二、 核心功能矩阵 (纯数据向)

#### 1. 多源识别引擎 (The Sensors)
*   **本地基线读取**：通过主程序 API 读取现有的 `artist`, `title`, `album`, `isrc`, `duration`。
*   **AcoustID 声纹探针**：获取 Chromaprint，请求 MusicBrainz（获取别名、Release 列表、ISRC）。
*   **国内文本源兜底**：网易云、QQ 音乐的文本搜索 API（用于声纹全挂时的抢救）。
*   ~~ACRCloud 云端声纹~~：暂不采用（付费服务，14天试用后 $49+/月）。

#### 2. 智能仲裁器 (The Brain) - *核心护城河*
*   **ISRC 绝对锚定**：本地 ISRC 匹配直接采信，跳过模糊搜索。
*   **时长惩罚机制**：时长差 > 5秒的候选直接淘汰（防同名异版误判）。
*   **加权投票算法**：歌名(50%) + 艺术家(35%) + 专辑(15%)，结合简繁统一转换计算相似度。
*   ~~双声纹交叉验证~~：暂不采用（ACRCloud 付费）。

#### 3. 富媒体增强 (The Enrichment)
*   **高精度 LRC 注入**：精准匹配国内源歌词，支持双语（中/英或中/罗马音）按时间轴合并，写入 `LYRICS` 标签。
*   **封面优选与画廊**：抓取多源封面，自动剔除低分辨率/带水印图片，提供"封面画廊"供用户挑选最完美的高清封面。

#### 4. API 调度与缓存 (The Shield)
*   **令牌桶限流**：严格遵循 API 请求频率限制，保护国内源不被封禁。
*   **24小时结果缓存**：基于 `MD5(artist+title)` 或文件 Hash 的本地 KV 缓存，避免重复消耗配额。

---

### 🎨 三、 极致的 UI/UX：可视化校对工作台

既然不需要处理文件移动，**"人工介入的体验"** 就是插件成败的关键。

#### 1. 扫描触发器 (Scanner)
*   **增量全库扫描**：基于数据库记录，只扫描未处理过或标签有变动的歌曲。
*   **目录/歌单定向扫描**：让用户右键某个专辑或歌单，选择"深度刮削此专辑"。

#### 2. "交通灯"分流看板
扫描结果自动分为三个 Tab：
*   🟢 **绿灯区 (自动已应用)**：置信度 ≥ 0.85。系统自动写回标签，列表展示修改日志（提供一键撤销）。
*   🟡 **黄灯区 (待人工校对)**：0.60 ≤ 置信度 < 0.85。**这是核心工作区**。
*   🔴 **红灯区 (刮削失败)**：置信度 < 0.60 或全网无结果。提供手动输入框或跳转外部搜索引擎的快捷链接。

#### 3. 沉浸式 Diff 校对面板 (黄灯区核心)
点击黄灯歌曲，展开类似 Git 代码对比的 Diff 面板：
*   **左侧 (当前标签)**：显示 Songloft 现有的 Artist/Title/Album/Cover。
*   **右侧 (系统推荐)**：显示仲裁器选出的最佳结果，并**标注数据来源与置信度**（如：`[AcoustID 92分]`）。
*   **操作**：
    *   一键 `Accept` (采纳推荐)。
    *   在推荐基础上微调（如修改错别字）。
    *   从"多候选画廊"中切换（例如系统推荐了《晴天 (Live)》，用户可切换回原版）。
    *   支持**批量勾选** -> **批量采纳系统推荐**。

### 🗄️ 四、 纯粹的数据流与状态机

没有文件 I/O 的干扰，数据流变得极其清晰：

```text
[用户触发扫描 / 主程序推送新歌入库事件] 
   ↓
[读取 Songloft 媒体库元数据] (包含 ISRC, 时长)
   ↓
[并行发起识别请求] (AcoustID + 文本源) 
   ↓ (受 API 缓存与限流器保护)
[智能仲裁器 (Arbitrator)] 
   ↓ (计算置信度，应用时长惩罚)
[写入插件本地 DB (scrape_status)]
   ├─→ 置信度 ≥ 0.85 ➔ 直接调用主程序 API 更新元数据 ➔ 记入绿灯区
   ├─→ 0.60 ≤ 置信度 < 0.85 ➔ 存入 pending_queue ➔ 渲染到黄灯区 UI
   └─→ 置信度 < 0.60 ➔ 记入红灯区
```

### 🗺️ 五、 纯数据向插件的敏捷迭代路线

#### 🚀 Phase 1: 核心引擎与基础 UI (MVP)
*   **目标**：跑通"读取 -> AcoustID + 文本搜索 -> 仲裁 -> 写回"的基础闭环。
*   **交付**：
    *   实现 `arbitrator.ts` (包含时长惩罚和基础加权投票)。
    *   实现 API 缓存与限流中间件。
    *   基础 UI：扫描进度条、简单的"成功/失败/待确认"列表。
    *   实现基础的自动写回（仅写 Artist/Title/Album）。

#### 🌟 Phase 2: 沉浸式校对 (核心壁垒)
*   **目标**：打造极致的黄灯区校对体验。
*   **交付**：
    *   集成 ISRC 锚定。
    *   上线"沉浸式 Diff 校对面板"，支持多候选切换。
    *   实现批量操作（批量采纳、批量忽略）。

#### 🎵 Phase 3: 富媒体增强与极客细节 (完全体)
*   **目标**：补全歌词、封面等富媒体数据，完善高级设置。
*   **交付**：
    *   高精度歌词匹配与写入（支持双语合并）。
    *   封面画廊（多候选封面展示与选择）。
    *   高级设置面板（调整权重、配置 API Key、清理缓存）。
    *   元数据健康度看板（展示全库缺失字段的比例）。

---

### 💡 总结

彻底砍掉文件操作后，插件的开发复杂度大幅降低，您可以将 100% 的精力投入到**仲裁算法的调优**和**Diff 面板的交互细节**上。这种"非侵入式"的纯数据增强插件，不仅完美契合 QuickJS 沙箱环境，更能成为 Songloft 生态中最硬核、最不可或缺的"官方级"组件。
